# SpeedCheck-CM

Simulation d’un système distribué pour analyser les défis de la connectivité Internet au Cameroun.

Ce projet utilise plusieurs services :
- Un coordinateur réseau (Flask)
- Des nœuds de stockage / mesure (Flask)
- Un client local (CLI)
- Un système de transfert distribué
- Un mécanisme de tolérance aux pannes
- Une architecture scalable

### Objectif général
Créer une simulation technique démontrant comment l’infrastructure numérique peut être distribuée pour améliorer la connectivité, la surveillance et le diagnostic des performances réseau au Cameroun.

### Technologies :
- Python
- Flask
- Requests
- Docker
- JSON API
- Architecture distribuée
